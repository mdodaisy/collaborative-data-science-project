Research Question

How does benthic cover vary across three reef zones of the Belize Barrier Reef?

Visualization of Data

1) Metadata

This dataset contains the metadata for quadrat photos taken every 5m along 50m transects such that ten photos were taken for each transect. Six transects were conducted at each of the three study sites in Belize: lagoon (whale shoal), fore reef, and back reef. The dataset contains the image name, site of collection, date of collection, transect number, and quadrat number.

2) Percent Cover Dataset

This dataset consists of percent cover data for each quadrat photo. Percent cover was calculated using CoralNet, a software that automates the analysis of coral reef benthic images. Essentially, CoralNet generates 25 random points over each image, identifies/labels the points, and then calculates percent cover from the point data. The dataset contains the image ID, image name (same as above), annotation status (whether the image has been labeled), number of collection points (25 points for each photo), and various columns corresponding to different benthic categories such as hard coral species, soft coral species, macroalgal species, etc. See the Glossary for a key to each category.

Data Analysis

Principal Coordinate Analysis

Principal Coordinate Analysis (PCoA) is a technique used to visualize multidimensional data. Since we have benthic cover data (functional group abundances from 0-1), we can use the Manhattan distance measure to calculate pairwise distances between each transect in our dataset. This will, ultimately, form a distance matrix that we can then analyze using PCoA to determine how similar or different the benthic cover of each transect is to one another.
Note: The Bray-Curtis distance measure could also be used for abundance data, but here we chose the Manhattan distance to analyze the absolute differences in percent cover, rather than relative metrics.

First, we need to split the metadata (containing the information of the sites and transects) and the community-matrix (containing the percent cover of each functional group on each transect) into two separate datasets before performing a PCoA.

# create metadata
benthic.meta <- benthic.wide %>%
  select(Habitat, Transect)

# create percent cover dataset
benthic.base <- benthic.wide %>%
  ungroup() %>%
  select(-Habitat, -Transect)

Next, we need to create our distance matrix. This can be done using the vegdist() function which calculates pairwise dissimilarities between pairs of transects. Essentially, this will yield the absolute difference in percent cover across all functional groups — ecologically, it will represent the “total amount of compositional change between two transects.”

# distance matrix using the Manhattan distance measure
benthic.dist <- vegdist(benthic.base, method = "manhattan")

The wcmdscale() function performs a Principal Coordinate Analysis. It takes the distance matrix and finds coordinates in a Euclidean space that best preserve those distances. k = 2 → projects into 2 dimensions (for plotting). eig = TRUE → returns eigenvalues (variance explained per axis). Unlike Principal Component Analysis (PCA) that works directly on Euclidean distances, PCoA accepts any distance matrix (e.g., Manhattan, Bray-Curtis).

benthic.pcoa.noscores <- wcmdscale(benthic.dist, k = 2, eig = TRUE)

Scores & Convex Hulls

Now, we need to extract the scores, or the coordinates, for two different components of the ordination: 1) the species scores: scores for the functional groups (columns in the distance matrix) and 2) the site scores: scores for the transects (rows in the distance matrix). The species loadings will be used to map the functional groups onto the PCoA to visualize functional group-site relationships while the site scores will be used to create convex hulls of each habitat. Convex hulls enclose the points from each habitat into a polygon to visualize group separation.

# extract species scores
benthic.pcoa <- BiodiversityR::add.spec.scores(benthic.pcoa.noscores, benthic.base, Rscale = TRUE, scaling = 1, multi = 1) # these are the coordinates for each functional group on the PCoA

# compile the species scores into a tidy dataset
benthic.species.scores <- as.data.frame(benthic.pcoa$cproj) %>%
  mutate(species = rownames(.))

Generalized Linear Model

Coral cover is a widely used indicator of reef condition, so we next examined the factors that may structure coral abundance across our transects. Here, we focused on two drivers with well-established roles in shaping coral recruitment and survival: macroalgae cover and reef habitat. Disturbances that kill corals create space for fast-growing macroalgae to take over and suppress coral recovery. Habitat context also matters, as the natural gradient from lagoon to back reef to fore reef is accompanied by shifts in water quality, wind and wave exposure, nutrient availability, etc.

As we can see from these two plots, there appears to be trends in the abundance of coral with macroalgae and reef habitat. So, we can use modeling to determine the significance of those trends.

To test whether we will need to use a general linear model or a generalized linear model, we must check the normality of coral abundance, our response variable.

benthic.wide %>%
  ggplot(aes(x= Coral)) +
  geom_histogram() +
  theme_bw() +
  xlab("Coral Abundance") +
  ylab("Count")

From the histogram, you can see that coral abundance is not normally distributed, so we must model our data using a different distribution. The beta distribution is appropriate for this as our abundance data is continuous from 0-1.

Running the model

We will perform a Bayesian Generalized Linear Model using the package {brms}. Our model, in this case, will consist of macroalgae cover and habitat as predictor variables and coral cover as our response variable. Bayesian modeling was chosen over frequentist modeling because it is well suited for small datasets with high variability. Bayesian models provide full posterior distributions and credible intervals that allow us to evaluate the magnitude and uncertainty of habitat and macroalgae effects on coral cover even with limited sample size.

Now, we will run a generalized linear model using the argument family = beta.

# run the model
if (file.exists("coral_model.rds")) {
  coral.glm <- readRDS("coral_model.rds")
} else {
  coral.glm <- brm(Coral ~ Habitat * Macroalgae,
                   data = cm.data,
                   family = "beta",
                   seed = 690)
}

Coral ~ Macroalgae

Now let’s look at solely the effects of macroalgae cover on coral cover.

Conclusions

The three reef zones examined in this study differ in their benthic community composition, such that the lagoon is dominated by seagrass and macroalgae whereas the back reef is characterized by coral, epilithic algal matrix (EAM), and cyanobacteria. Despite these clear community differences, the model detected no measurable effects of habitat, macroalgae cover, or their interaction on coral cover. This likely reflects the limited statistical power of our small sample size (n = 18). Therefore, while benthic assemblages vary greatly across reef zones in Belize, a larger dataset may be needed to reliably evaluate the influence of habitat and macroalgae on coral cover across reef zones in Belize.
