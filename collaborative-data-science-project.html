Research Question

How does benthic cover vary across three reef zones of the Belize Barrier Reef?

Six transects were conducted at each of the three study sites in Belize: lagoon (whale shoal), fore reef, and back reef. The dataset contains the image name, site of collection, date of collection, transect number, and quadrat number.

2) Percent Cover Dataset

This dataset consists of percent cover data for each quadrat photo. Percent cover was calculated using CoralNet, a software that automates the analysis of coral reef benthic images. Essentially, CoralNet generates 25 random points over each image, identifies/labels the points, and then calculates percent cover from that data. The dataset contains the image ID, image name (same as above), annotation status (whether the image has been labeled), number of collection points, and various columns corresponding to different benthic categories such as hard coral species, soft coral species, macroalgal species, etc. See the Glossary for a key to each category.

Now, we need to split the metadata (containing the information of the sites and Transect) and the community-matrix (containing the cover percentage of each functional group on each transect) before performing a Principal Coordinate Analysis (PCoA).

library(dplyr)
library(vegan)
library(ggplot2)
library(tibble)

# Split metadata and community matrix- so that only imporatant metadata remains (site code and habitat type.)
benthic.meta <- benthic.wide %>%
  select(Site, Transect)

benthic.base <- benthic.wide %>%
  select(-Site, -Transect)

The vegdist() calculates pairwise dissimilarities between pairs of transects. It will yield the absolute difference in cover across all functional groups:: Ecologically it will represent the “Total amount of compositional change between two transects”

The wcmdscale performs a Principal Coordinate Analysis:: It takes the distance matrix and finds coordinates in a Euclidean space that best preserve those distances. k = 2 → project into 2 dimensions (for plotting). eig = TRUE → returns eigenvalues (variance explained per axis). Unlike a PCA (Principal Component Analysis), the PCoA handles any distance matrix (manhattan, Bray-Curtis). PCA only handles Euclidian distances.

#  Distance matrix using a Manhattan distance: 
benthic.base <- benthic.wide[-c(1:3)]
benthic.dist <- vegdist(benthic.base, method = "manhattan")
# Then perform PCoA with the wcmdscale() function from the vegan package
benthic.pcoa.noscores <- wcmdscale(benthic.dist, k = 2, eig = TRUE)
